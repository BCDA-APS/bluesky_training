Time Series#

Record a series of readings over time.

Objective

In this notebook, we show how to record time series with several examples:

  • scaler

  • temperature

  • multiple detectors

See Count the scaler for more details about configuring and using a scaler.

Time Series: scaler1#

First, set the scaler’s counting time to 0.2 seconds when we count here.

NOTE

The scaler used here is an EPICS software simulator of real scaler hardware. It has a bug that almost always adds to the configured preset time. Expect the counting times to be 0.3s in this document.

[2]:

scaler1.preset_time.put(0.2)

Now, use the bluesky RunEngine (RE) to count scaler1. We’ll use the bluesky plan (bp) called count().

The first argument to bp.count() is a list of detectors. At each reading, each “detector” is triggered (to executes its data acquisition steps). Once all detectors have finished triggering, all detectors are read out and reported.

Any “detector” (readable device or signal) can be recorded as a time series. For example scaler1 (in this simulator) is such a readable device. A readable device or signal has a .read() method that returns time-stamped data.

Here, we read scaler1.

[3]:

scaler1.read()

Using scaler1, record 10 measurements at 1.0 second intervals.

We choose an interval time that is longer than the count time for the scaler.

[4]:

RE(bp.count([scaler1], num=10, delay=1.0))

Time Series: multiple detectors#

Measure a time series with a list of several “detectors”.

It is possible to use a component of a device as a detector. Here, we pick the simulated double-crystal monochromator energy (dcm.energy) as a detector. See the XAFS scan example for more details about the dcm.energy device.

See the lineup 1D HOWTO for more details about the noisy signal.

[7]:

RE(bp.count([temperature, noisy, dcm.energy, scaler1], num=10, delay=1.0))
